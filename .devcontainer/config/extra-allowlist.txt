# Extra firewall allowlist for the Claude sandbox.
#
# One hostname per line. '#' starts a comment. Edit this file on the HOST, then
# re-apply with:   docker exec claude-code sudo /usr/local/bin/init-firewall.sh
# (or restart the container). Mounted read-only at
# /etc/claude-firewall/extra-allowlist.txt.
#
# These supplement the always-on allowlist baked into init-firewall.sh. Use this
# instead of editing the script. For fully open egress, set FIREWALL_MODE=permissive.
#
# --- Examples (uncomment to enable) ---
# crates.io                 # runtime `cargo install`
# index.crates.io
# static.crates.io
# deb.debian.org            # runtime `apt-get install`
# security.debian.org
# huggingface.co            # ML model/dataset downloads
# developer.mozilla.org     # docs for Claude WebFetch
# stackoverflow.com
#
# Note: github.com, api.github.com, and the release-asset hosts are ALWAYS
# allowed by init-firewall.sh — you do not need to list them here.
#
# --- AWS ---
# AWS endpoints are wildcard, per-region, and CloudFront-fronted, so a hostname
# line here can't reach them (a bare `amazonaws.com` has no useful A record). Use
# the directive below: it loads AWS's published CIDR feed (ip-ranges.json, the
# AWS analog of GitHub's /meta) — covering `aws sso login` + the CLI (oidc /
# portal.sso / sts / s3 / awsapps.com). Optionally narrow by region to shrink the
# ipset, e.g. `@aws-ip-ranges us-east-1 eu-west-1` (GLOBAL/CloudFront always kept).
